2026 年 5 月 7 日、OpenAI が Realtime API に **3 つの新モデル** （`gpt-realtime-2` / `gpt-realtime-translate` / `gpt-realtime-whisper`）を一気に投入した。前世代 `gpt-realtime` から context window が 4 倍 (32K → 128K) になり、GPT-5 級の reasoning を音声ターンに内蔵した、いわゆる「考えながら喋る」モデルが商用 API に降りてきた格好だ。

ほぼ同じ時期に Cartesia / ElevenLabs / Deepgram も次世代モデルを並べ、オープン側でも Moshi、Sesame CSM、irodori-TTS が出揃った。本稿では 2026-05 時点の **TTS / STT / 双方向 (speech-to-speech, S2S)** を同じ粒度で並べた survey を残しておく。

## まず 3 つの軸で景色を整理する

選定の判断軸は次の 3 つに集約される。本稿の比較もこの軸で切る:

1. **レイテンシ** — リアルタイム対話に使えるのは概ね **300 ms 以下**、UX 上は **〜100 ms** が会話の閾値とされる
2. **双方向性** — 単方向 TTS / STT か、それとも full-duplex (聴きながら喋る) S2S か
3. **提供形態** — クラウド API か、オープンウェイトでオンプレ・オンデバイスか

「リアルタイム対応」と謳うモデルでも、サーバ往復・ネットワーク・前処理を含めた **end-to-end** で見ると 300 ms を切るのは案外少ない。表記が `モデル単体レイテンシ` か `TTFA (Time To First Audio)` か `end-to-end` かは要注意。

## TTS 系

### OpenAI gpt-realtime-2（TTS としても）

- リリース: 2026-05-07
- 提供: OpenAI Realtime API (`wss://api.openai.com/v1/realtime?model=gpt-realtime-2` / WebRTC)
- 言語: 多言語（Translate モデルは入力 70+ → 出力 13 言語）
- 価格: 音声入力 **$32 / M tokens**（キャッシュ済み $0.40 / M）、出力 **$64 / M tokens**
- ライセンス: 商用 API のみ
- 何が新しいか: 「考えるあいだの沈黙」が消えた。reasoning effort を `minimal / low / medium / high / very-high` で切り替えられ、ツール呼び出し中も "checking your calendar" のように口頭で進捗を漏らす設計が追加された。

S2S にも TTS にも STT にも使える "総合モデル" としての立ち位置で、`gpt-realtime` 系の中核は完全にここに移った。

### Cartesia Sonic-3 / Sonic Turbo

- リリース: Sonic-3 が 2025 後半、Sonic Turbo の超低遅延変種が継続提供
- 提供: WebSocket streaming API、SDK 多数
- 言語: **40+ 言語**（9 言語のインド語族を含む）
- レイテンシ: 公称 **モデル単体 < 100 ms**（"次点の 4 倍速い" と表現）／別箇所では **~90 ms** のクレーム
- 価格: 約 **$0.030 / 分**相当 (実体は 1 credit = 1 文字課金)
- ライセンス: 商用 API、SOC 2 Type II / HIPAA / PCI Level 1 対応
- 何が新しいか: state-space model 系の流れを汲み、**笑い・感情・抑揚** を含めた表現を低遅延のまま吐ける。10 秒録音から voice cloning。

「最低レイテンシで安定 TTS」を取りに行く用途では現状の本命。

### ElevenLabs (Eleven v3 / Flash v2.5)

- リリース: Eleven v3 が flagship。Flash v2.5 が低遅延変種
- 提供: REST + WebSocket streaming
- 言語: v3 が **70+ 言語**、Flash v2.5 が 32 言語
- レイテンシ: **Flash v2.5 ≈ 75 ms**（モデル単体）、Turbo v2.5 は 250–300 ms と段違いに遅く、現在は Flash 推奨
- 価格: 文字単価 (プランによる)
- ライセンス: 商用 API
- 何が新しいか: 5,000 文字超のロングフォームや感情表現は v3、レイテンシ命なら Flash v2.5、と **用途で明確に分かれた**。Conversational AI で agent オーケストレーション層も提供。

Scribe v2 Realtime（STT, ~150 ms, 90+ 言語）も同社が出していて、ElevenLabs だけで TTS + STT を組める。

### Deepgram Aura-2

- リリース: Aura-2 が 2025 後半、2026-04 に pronunciation control update
- 提供: REST + streaming WebSocket
- 言語: 多言語 (TTS) + 36+ 言語 (STT 側 Nova-3)
- レイテンシ: 200 ms 級
- 価格: TTS **$0.030 / 1k 文字**、STT (Nova-3 streaming) **$0.0048 / 分**、Voice Agent API **$0.075 / 分 (= $4.50 / 時)**
- ライセンス: 商用 API
- 何が新しいか: 単独で `Voice Agent API`（STT + TTS + LLM 呼び出しを統合した bidirectional wrapper）を **時間課金で透明** に出しているのは現状ここくらい。

### irodori-TTS-500M-v2（日本語特化・OSS）

- リリース: 2026 年
- 提供: Hugging Face `Aratako/Irodori-TTS-500M-v2`（ローカル推論）
- 言語: **日本語のみ**
- レイテンシ: ローカル推論ベース（CPU で約 500 ms、GPU でほぼリアルタイム）
- 価格: 無料 (OSS)
- ライセンス: **MIT**
- 何が新しいか: **絵文字を埋め込むだけで感情・話速・効果音を制御** できる Rectified Flow Diffusion Transformer (RF-DiT) 系の日本語 TTS。約 500M パラメータ、zero-shot voice cloning にも対応。

OpenAI や ElevenLabs が日本語をマルチリンガル枠で扱うのに対し、これは「日本語を一級市民として鍛えた」モデル。オンプレ・無制限生成・MIT を全部満たす日本語 TTS は他に並ばない。

## STT 系

### OpenAI gpt-realtime-whisper

- リリース: 2026-05-07
- 提供: Realtime API (WebSocket / WebRTC)。 **`/v1/audio/transcriptions` ではない** ことに注意
- 言語: Whisper 系譜の多言語サポート
- レイテンシ: streaming partial transcript（lat / acc のトレードオフを開発者側で調整可）
- 価格: **$0.017 / 分**
- 何が新しいか: 旧 `whisper-large-v3` が batch 前提だったのに対し、**音声が来た瞬間に部分書き起こしを返す** 設計。会議キャプション・コールセンター・voice agent の "ユーザーが何を言ったかを継続的に把握する" ループに直接使える。

### OpenAI gpt-realtime-translate（STT 派生枠）

5/7 リリースの 3 モデル目で、survey 上は STT の派生。**70+ 言語 → 13 出力言語** にライブ翻訳しながら音声を吐ける。価格 **$0.034 / 分**。同時通訳エージェントの土台になる。

### Deepgram Nova-3

- レイテンシ: 300 ms 級 streaming
- 言語: 36+
- 価格: **$0.0048 / 分**（streaming, Pay As You Go）
- 強み: 騒音下の認識精度と、Voice Agent API への組み込みやすさ。

### 補助: バッチ向けは insanely-fast-whisper

リアルタイム性が不要で「150 分の録音をとにかく速く一発で文字起こし」したいなら、過去記事の [insanely-fast-whisper](/blogs/posts/2026/03/2026-03-25-insanely-fast-whisper/) ルートが今でも有効。Flash Attention 2 + バッチで A100 上 98 秒。

## 双方向 (full-duplex / speech-to-speech) 系

この領域こそが 2026-05 の本丸。聴く・考える・喋るを 1 つの session 上で同時並行する設計の本気度を、各陣営が一気に出してきた。

### OpenAI gpt-realtime-2（再掲）

- フル機能の S2S。reasoning effort をミリ秒単位で調整できる
- 中断 (barge-in)、並列ツール呼び出し、長文 context (128K) を 1 session で保持
- 完全マネージドなので、自前で audio codec / VAD / 同時通話 buffer を握る必要がない

商用品質と reasoning 深さを両立したい場合、現状ここがデフォルト。

### Kyutai Moshi（OSS S2S）

- リポジトリ: [kyutai-labs/moshi](https://github.com/kyutai-labs/moshi)
- レイテンシ: 理論値 **160 ms**、実測 200 ms（L4 GPU）
- ライセンス: コード MIT (Python) / Apache (Rust)、モデル重み **CC-BY 4.0**
- on-device: `moshi_mlx/` 経由で iPhone / Mac (Apple Silicon) で量子化 (int4 / int8 / bf16) 推論
- 強み: **オープンソースで本物の full-duplex** が動く。会話を 2 ストリーム (user / assistant) として同時に流す Moshi 独自の設計

「プライバシー込みで音声 agent を握りたい」「自社モデルに置き換えたい」用途では現実的な選択肢。

### Sesame CSM (Conversational Speech Model)

- リリース: 2025-05（Hugging Face Transformers 4.52.1+ に統合）
- 提供: OSS、リポジトリ [SesameAILabs/csm](https://github.com/SesameAILabs/csm)
- 言語: 英語中心（他言語は data contamination 由来で動かなくはないが推奨されないと README に明記）
- レイテンシ: README に明示なし
- 価格: 無料 (Apache-2.0)
- 構成: **Llama backbone + Mimi audio decoder**（Moshi と同じ Mimi codec）
- 何が新しいか: **会話履歴（音声を含む）を条件として prosody を決める**。これにより、行を読み上げるだけの TTS よりターン間が自然になる。

### Hume EVI 3（感情解釈付き S2S）

- リリース: **2025-05-29**（公式: [introducing-evi-3](https://www.hume.ai/blog/introducing-evi-3)）
- 提供: API、iOS アプリ、ライブデモ
- 言語: 英語中心（French / German / Italian / Spanish を順次強化）
- レイテンシ: 公称 **< 300 ms**（最適環境）
- 価格: 非公開（商用）
- 何が新しいか: 30 種類の感情・スタイルを「明示的または暗黙的に」生成し、ユーザー音声からも **9 種の感情カテゴリのうち 8 種を識別** する。100,000+ カスタムボイス。「電話オペレーターの感情まで読んで返したい」「Voice UX を表情豊かにしたい」用途で唯一無二。

### Google Gemini Live API（マルチモーダル S2S）

- ドキュメント: [Live API overview](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/live-api)
- バックボーン: **Gemini 2.5 Flash Native Audio**
- 言語: **70 言語**
- 接続: WebSocket bidi-streaming で audio + video + text を同時に流せる
- 強み: **音声と画像を同じ session に乗せた S2S** はここの独擅場。AR グラスやカメラ越しの voice agent では事実上選択肢が無い

### Deepgram Voice Agent API（オーケストレーション層）

- 形式: STT + LLM 連携 + TTS を 1 endpoint に束ねたラッパ
- 価格: **$0.075 / 分**（Pay As You Go） = 約 $4.50 / 時 で **LLM 利用料込み**
- 立ち位置: 自分で Pipecat / LiveKit Agents を組まなくても、固定価格・固定品質で voice agent を立てられる

## 選び方の指針

**日本語の表現品質を最優先する**
→ オンプレなら **irodori-TTS-500M-v2**、クラウドなら **OpenAI gpt-realtime-2** か **Gemini Live**。ElevenLabs v3 / Eleven Multilingual v2 も日本語は通るがイントネーション差は実機テスト推奨。

**とにかく短い TTFA（電話オペレーター・音声入力 UI 等）**
→ **ElevenLabs Flash v2.5**（~75 ms）か **Cartesia Sonic-3 / Sonic Turbo**（<100 ms）。両者は感情表現と価格モデルで分かれる。

**双方向対話 (full-duplex / 聴きながら喋る) が必須**

- クラウドで完結: **gpt-realtime-2**
- 自前運用 (OSS): **Moshi**
- 文脈依存の prosody を取りに行く: **Sesame CSM**
- 感情理解まで欲しい: **Hume EVI 3**
- マルチモーダル (画像 + 音声): **Gemini Live**（事実上ここ一択）

**オンプレ・on-device・OSS が条件**
→ **Moshi**（フルダックス）、**Sesame CSM**（文脈条件付き音声生成）、**irodori-TTS**（日本語 TTS）の組み合わせ。バッチ STT は引き続き [insanely-fast-whisper](/blogs/posts/2026/03/2026-03-25-insanely-fast-whisper/)。

**voice agent をパッケージで運用したい（自分で繋ぎ込みたくない）**
→ **Deepgram Voice Agent API** か **ElevenLabs Conversational AI**。価格は明朗、品質は LLM 選定に依存。

## まとめ

2026-05 時点で「次の一手」を引いたのは間違いなく OpenAI の Realtime 3 兄弟だが、低レイテンシ TTS は Cartesia / ElevenLabs、商用 voice agent は Deepgram、full-duplex OSS は Moshi、文脈付き音声は Sesame CSM、感情は Hume、マルチモーダルは Gemini Live、日本語は irodori、と **各役割の "1 番手" が見えやすくなったのが本当の更新点** だと整理できる。

「voice agent を作るなら全部 OpenAI」という時期はもう終わって、用途に合わせて束ねる段階に入った。
