## 概要

AIアフィリエイト自動化は、AIエージェントを使って SNS 投稿、ネタ収集、コンテンツ生成、投稿スケジューリング、クリック・収益分析を回す収益化パターン。Claude Code のようなコーディングエージェントを、コード生成だけでなく「小さなマーケティング運用システムを組む実行環境」として使う点が特徴。大量投稿だけを目的にすると凍結・品質低下・法令違反のリスクが高く、媒体規約と広告表記を前提にした運用設計が必要になる。

## 基本フロー

1. RSS、SNS トレンド、競合投稿、商品データから投稿ネタを収集する。
2. AI がテーマ、投稿文、画像プロンプト、ハッシュタグ、CTA を生成する。
3. ASP や物販アフィリエイトのリンクを、媒体要件に沿って挿入する。
4. X、Instagram、TikTok、YouTube Shorts などに投稿する。
5. クリック率、成果、エンゲージメントを収集し、次の投稿に反映する。

## ASP の役割

ASP（Affiliate Service Provider）は、広告主とメディア運営者を仲介する基盤。案件カタログ、トラッキングリンク、成果承認、不正検知、報酬支払いをまとめて提供する。AI 自動化側から見ると、ASP は「紹介する案件を探す場所」だけではなく、収益計測と支払いを外部化するインフラでもある。

## 典型的な組み合わせ

| 媒体 | 向いている案件 | 注意点 |
|------|----------------|--------|
| X | SaaS、ガジェット、金融、学習教材 | 自動投稿・重複投稿の規約リスク |
| Instagram | 美容、健康、ファッション、楽天 ROOM | 画像・表記・ステマ規制への配慮 |
| TikTok | アプリ CPI、EC、短尺動画向け商品 | 誇張表現とアカウント品質管理 |
| YouTube Shorts | 転職、金融、SaaS、教育 | 概要欄導線と広告表記の明確化 |
| ブログ + SNS | SEO と SNS のハイブリッド | 記事品質と独自性の維持 |

## 運用リスク

- **SNS 規約**: 主要 SNS は大量自動投稿や不自然な自動化を制限している。API 利用条件、投稿頻度、重複コンテンツ制限を確認する。
- **広告表記**: 日本ではアフィリエイト広告を含むことを明示する必要がある。AI 生成でも表示義務は変わらない。
- **品質管理**: AI 生成のまま大量投稿すると、誤情報、低品質、同質化が起きやすい。人間レビューや投稿前チェックを組み込む。
- **ASP の媒体要件**: SNS 直投稿が不可の案件、運営サイト必須の案件、高単価だが審査が厳しい案件がある。
- **収益再現性**: 成果はニッチ、アカウント信頼度、案件、導線、競合状況に左右される。

## Claude Code との関係

[Claude Code](/blogs/wiki/tools/claude-code/) は、投稿生成そのものよりも、運用パイプラインの構築に向いている。たとえばデータ収集スクリプト、投稿テンプレート、スケジューラー、分析レポート、A/B テスト用のバリエーション生成などを組み合わせることで、マーケティング業務を半自動化できる。
